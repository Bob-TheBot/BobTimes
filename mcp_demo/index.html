What is the Model Context Protocol?

MCP is an open protocol that standardizes how applications provide context to Large Language Models (LLMs). Think of it as the "USB-C for AI" - a universal standard that enables seamless communication between AI models and various data sources.

✓

Standardized Communication

Uses JSON-RPC 2.0 for consistent message exchange

✓

Multiple Transport Options

Supports stdio, SSE, and Streamable HTTP

✓

Bidirectional Communication

Both clients and servers can initiate requests

Advanced Feature: Sampling

Sampling allows MCP servers to request LLM completions through the client, enabling servers to access "intelligence" for their tasks.

1

2

3

Transport Layer

Understanding MCP's communication methods and choosing the right transport for your use case

Transport Options

MCP offers three main transport methods, each optimized for different use cases

Streamable HTTP

RECOMMENDED

The modern standard for remote connections, introduced in March 2025. Simplifies communication through a single HTTP endpoint.

✓ Single HTTP endpoint
✓ Easy to implement
✓ Future-proof
# FastMCP Configuration mcp.run( transport="streamable-http", host="0.0.0.0", port=8001 )

Standard I/O (stdio)

ACTIVE

Ideal for local communication, command-line tools, and connecting programs on the same computer via stdin/stdout.

✓ Simple & direct
✓ Local communication
✓ CLI integration
# Client Configuration { "command": "npx", "args": ["-y", "@upstash/context7-mcp"] }

Server-Sent Events

DEPRECATED

Previously used for remote connections. Being phased out in favor of Streamable HTTP for better simplicity and performance.

⚠ Two communication channels
⚠ More complex setup
⚠ Being deprecated
# Legacy - Not Recommended mcp.run( transport="sse" )

Communication Lifecycle

MCP follows a three-phase communication pattern

🚀

1. Initialization

Client sends initialize request, server responds with capabilities, client acknowledges with initialized notification.

🔄

2. Message Exchange

Bidirectional communication where both client and server can send requests and notifications.

🛑

3. Termination

Either party can send a close command to gracefully terminate the connection.

Tools: Executable Functions

Tools are functions that MCP servers expose to clients, allowing LLMs to perform actions in the real world. They're the "hands" of your AI assistant.

• Database Operations: Query, insert, update data
• API Calls: Interact with external services
• File Operations: Read, write, manipulate files
• Calculations: Perform complex computations
# FastMCP Tool Example @mcp.tool() def calculate_sum(a: float, b: float) -> float: """Calculate the sum of two numbers.""" return a + b @mcp.tool() def get_weather(city: str) -> dict: """Get current weather for a city.""" # API call implementation return weather_data
📄

Resources: Data Providers

Resources provide read-only access to data and content. They're the "knowledge base" that gives context to your AI assistant.

• Static Resources: Fixed content like documentation
• Dynamic Resources: Real-time data like logs
• Subscriptions: Get notified of changes
• URI-based: Addressable via unique identifiers
# FastMCP Resource Example @mcp.resource("logs://application") def get_app_logs() -> str: """Get application logs.""" with open("app.log") as f: return f.read() @mcp.resource("customer://{{customer_id}}/logs") def get_customer_logs(customer_id: str) -> str: """Get logs for specific customer.""" return load_customer_logs(customer_id)
💬

Prompts: Reusable Templates

Prompts are reusable templates that help structure interactions with LLMs. They provide consistent, well-crafted instructions for specific tasks.

• Parameterized: Accept dynamic inputs
• Consistent: Standardized formatting
• Reusable: Share across applications
• Discoverable: Listed by servers
# FastMCP Prompt Example @mcp.prompt("analyze-issue") def analyze_customer_issue( customer_id: str, issue_type: str = "general" ) -> str: """Generate analysis prompt for customer issues.""" return f""" Analyze the following customer issue: Customer ID: {customer_id} Issue Type: {issue_type} Please provide: 1. Root cause analysis 2. Recommended solutions 3. Prevention strategies """

Code Comparison: Raw MCP vs FastMCP

Raw MCP Implementation

# Raw MCP - 50+ lines of boilerplate import asyncio from mcp.server import Server from mcp.types import Tool, TextContent server = Server("calculator") @server.list_tools() async def list_tools(): return [ Tool( name="calculate_sum", description="Calculate sum of two numbers", inputSchema={ "type": "object", "properties": { "a": {"type": "number"}, "b": {"type": "number"} }, "required": ["a", "b"] } ) ] @server.call_tool() async def call_tool(name: str, arguments: dict): if name == "calculate_sum": a = arguments["a"] b = arguments["b"] result = a + b return [TextContent( type="text", text=f"Result: {result}" )] # Transport setup, error handling, etc...

FastMCP Implementation

# FastMCP - Just 8 lines! from mcp.server.fastmcp import FastMCP mcp = FastMCP("Calculator") @mcp.tool() def calculate_sum(a: float, b: float) -> float: """Calculate the sum of two numbers.""" return a + b if __name__ == "__main__": mcp.run() # That's it! Schema, validation, transport - all handled automatically

Real-World Example: Customer Support Server

This example demonstrates a complete MCP server for customer support operations with tools, resources, and prompts.

Server Implementation

from mcp.server.fastmcp import FastMCP import sqlite3 mcp = FastMCP("CustomerSupport") # Tool: Get support tickets @mcp.tool() def get_support_tickets( customer_id: str, status: str = "open" ) -> list: """Get support tickets for customer.""" conn = sqlite3.connect("support.db") cursor = conn.execute( "SELECT * FROM tickets WHERE customer_id=? AND status=?", (customer_id, status) ) return cursor.fetchall() # Resource: Application logs @mcp.resource("logs://application") def get_app_logs() -> str: """Get application logs.""" with open("app.log") as f: return f.read() # Prompt: Issue analysis @mcp.prompt("analyze-issue") def analyze_issue( customer_id: str, issue_description: str ) -> str: """Generate issue analysis prompt.""" return f""" Analyze this customer issue: Customer: {customer_id} Issue: {issue_description} Provide: 1. Root cause analysis 2. Solution steps 3. Prevention measures """

Client Usage

# Connect to MCP server client = SimpleMCPClient() await client.connect_to_remote_server( "http://localhost:8001/mcp" ) # Use tools via natural language response = await client.process_query( "Get all open tickets for customer CUST001" ) # Access resources directly logs = await client.get_resource( "logs://application" ) # Use prompts with parameters analysis_prompt = await client.get_prompt( "analyze-issue", customer_id="CUST001", issue_description="Login failures" ) # Run prompt with Claude await client.run_prompt_with_claude( analysis_prompt )

Best Practices

🔧 Tool Design

  • • Keep tools focused and single-purpose
  • • Use clear, descriptive names and docstrings
  • • Validate inputs and handle errors gracefully
  • • Return structured data when possible

📄 Resource Management

  • • Use meaningful URI schemes
  • • Implement caching for expensive operations
  • • Support subscriptions for dynamic content
  • • Handle large resources efficiently
